# User Service 业务规则

## 1. 核心业务功能

### 1.1 用户注册
- **功能描述**：用户通过手机号和密码注册账号
- **业务规则**：
  - 手机号必须符合中国大陆手机号格式（11位数字，以1开头）
  - 密码长度至少6位，最多20位
  - 密码必须包含至少一个字母和一个数字
  - 每个手机号只能注册一个账号
  - 注册时需要发送验证码进行验证

### 1.2 用户登录
- **功能描述**：用户通过手机号和密码登录系统
- **业务规则**：
  - 手机号和密码不能为空
  - 登录失败超过5次，账号将被锁定30分钟
  - 登录成功后生成Token，有效期24小时
  - 支持记住密码功能，但只在安全设备上使用

### 1.3 用户信息管理
- **功能描述**：用户查看和修改个人信息
- **业务规则**：
  - 用户名长度2-20个字符
  - 头像支持JPG、PNG格式，大小不超过2MB
  - 生日必须是过去的日期
  - 修改敏感信息需要验证密码

### 1.4 密码管理
- **功能描述**：用户重置密码、修改密码
- **业务规则**：
  - 修改密码需要验证旧密码
  - 重置密码需要通过手机号验证码验证
  - 新密码不能与旧密码相同
  - 密码修改后需要重新登录

| id | Long | PRIMARY KEY | 用户ID |

| password | String | NOT NULL | 加密后的密码 |
| username | String | NOT NULL | 用户名 |
| avatar | String | | 头像URL |
| birthday | Date | | 生日 |
| gender | Integer | | 性别（0：未知，1：男，2：女） |
| email | String | | 邮箱 |
| createTime | Timestamp | NOT NULL | 创建时间 |
| updateTime | Timestamp | NOT NULL | 更新时间 |
| lastLoginTime | Timestamp | | 最后登录时间 |
| status | Integer | NOT NULL | 状态（0：正常，1：锁定，2：禁用） |

| expireTime | Timestamp | NOT NULL | 过期时间 |
| status | Integer | NOT NULL | 状态（0：未使用，1：已使用） |

## 3. API 接口

### 3.1 注册接口
- **URL**：`/api/user/register`
- **方法**：POST
- **请求参数**：
  ```json
  {
    "phoneNumber": "13800138000",
    "password": "Password123",
    "code": "123456",
    "username": "张三"
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "注册成功",
    "data": {
      "userId": 1,
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
  }
  ```

### 3.2 登录接口

## 4. 业务流程

### 4.1 注册流程
1. 用户输入手机号，点击获取验证码
2. 系统验证手机号格式是否正确
3. 系统生成验证码，发送到用户手机
4. 用户输入验证码、密码、用户名
5. 系统验证验证码是否有效
6. 系统验证手机号是否已注册
7. 系统加密密码
8. 系统创建用户记录
9. 系统生成Token并返回

### 4.2 登录流程
1. 用户输入手机号和密码
2. 系统验证手机号和密码是否为空
3. 系统查询用户信息
4. 系统验证用户状态是否正常
5. 系统验证密码是否正确
6. 系统更新最后登录时间
7. 系统生成Token并返回用户信息

### 4.3 修改密码流程
1. 用户输入旧密码和新密码
2. 系统验证旧密码是否正确
3. 系统验证新密码格式是否正确
4. 系统验证新密码是否与旧密码相同
5. 系统加密新密码
6. 系统更新用户密码
7. 系统返回修改成功信息

### 4.4 重置密码流程
1. 用户输入手机号，点击获取验证码
2. 系统验证手机号是否已注册
3. 系统生成验证码，发送到用户手机
4. 用户输入验证码和新密码
5. 系统验证验证码是否有效
6. 系统验证新密码格式是否正确
7. 系统加密新密码
8. 系统更新用户密码
9. 系统返回重置成功信息

## 5. 安全规则

### 5.1 密码安全
- 密码使用 bcrypt 算法加密存储
- 密码传输过程中使用 HTTPS 加密
- 系统不存储明文密码
- 密码强度需要进行检测

### 5.2 验证码安全
- 验证码有效期5分钟
- 每个手机号每分钟最多发送1条验证码
- 验证码使用6位数字
- 验证码只能使用一次

### 5.3 Token安全
- Token使用 JWT 格式
- Token 有效期24小时
- Token 包含用户ID、过期时间等信息
- Token 泄露后可以通过修改密码使其失效

### 5.4 接口安全
- 所有接口需要进行参数校验
- 敏感接口需要进行权限验证
- 防止SQL注入、XSS攻击等安全漏洞
- 接口访问频率限制

### 5.5 数据安全
- 敏感数据脱敏处理
- 数据传输加密
- 定期数据备份
- 数据恢复机制

## 6. 错误处理

### 6.1 错误码
| 错误码 | 描述 |
|-------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 1001 | 手机号格式错误 |
| 1002 | 手机号已注册 |
| 1003 | 手机号未注册 |
| 1004 | 验证码错误 |
| 1005 | 验证码过期 |
| 1006 | 密码错误 |
| 1007 | 账号被锁定 |
| 1008 | 账号被禁用 |
| 1009 | 密码长度不足 |
| 1010 | 密码格式错误 |
| 1011 | 新密码与旧密码相同 |
| 1012 | 验证码发送过于频繁 |

### 6.2 错误信息格式
```json
{
  "code": 1001,
  "message": "手机号格式错误",
  "data": null
}
```

## 7. 性能要求

### 7.1 响应时间
- 登录接口响应时间不超过 500ms
- 注册接口响应时间不超过 800ms
- 其他接口响应时间不超过 300ms

### 7.2 并发处理
- 系统支持每秒至少 1000 个登录请求
- 系统支持每秒至少 500 个注册请求
- 系统支持同时在线用户数至少 10 万

### 7.3 可用性
- 系统可用性达到 99.9%
- 计划内维护时间不超过每月 4 小时
- 故障恢复时间不超过 30 分钟
